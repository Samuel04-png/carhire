S E C T I O N  1
Design System
Every visual decision in this system follows the design system defined here. No component is 
built with ad-hoc styles. All colors, typography, spacing, and motion rules are defined as design 
tokens and referenced consistently across the entire platform.
1.1  Brand Identity & Visual Direction
The Shark Car Hire brand communicates power, reliability, and premium quality. The visual 
language is clean, bold, and professional — dark backgrounds with sharp white type and 
electric blue accents. Think premium automotive meets corporate confidence. Inspired by 
brands like Hertz Gold, Enterprise Business, and BMW Digital — not generic template car hire 
sites.
Design Personality
• Bold but not aggressive — confident, not flashy
• Premium but accessible — high-end feel, not intimidating
• Trust-first — every design decision reinforces reliability and professionalism
• African pride — subtle Zambian/African identity woven in without clichés
• Mobile-first always — designed for a phone, enhanced for desktop

2.4  Booking Flow
The booking flow is a multi-step wizard. Each step is a separate view but no page reload — 
smooth animated transitions between steps. Progress bar at top shows Step 1 of 4. User can go 
back to previous steps. Form state persisted in Zustand so refreshing doesn't lose data.
S C R E E N  1 1   Booking Step 1 — Trip Details  [ /book/step-1 ]
• Selected vehicle summary card at top (photo, name, price/day) — always visible
• Fields: Pickup Location (dropdown + 'Enter custom address' option), Pickup Date, Pickup 
Time, Return Date, Return Time
• With Driver toggle: Self-Drive / With Chauffeur
• If airport transfer: show 'Flight Number' field + 'Arrival/Departure' toggle
• Duration + total price auto-calculated and displayed live as user selects dates
• Validation: pickup must be 2+ hours from now, return must be after pickup
• 'Continue' button disabled until all required fields valid

Admin & Operations Dashboard
The admin dashboard is a separate authenticated application accessible only to Shark Car Hire 
staff. It is the operational nerve centre — every booking, vehicle, client, driver, and revenue 
figure is visible and manageable from here. It runs at admin.sharkcarhire.org or /admin with 
role-based access control.
Admin dashboard shares the design system tokens but uses a lighter theme: white backgrounds, 
navy/blue accents. More data-dense than the public site. Built for desktop-first but fully functional on 
tablet.

S E C T I O N  8
Developer Notes & Rules
8.1  Code Quality Standards
Non-Negotiable Rules for Every Developer on This Project
• TypeScript strict mode. No any types. Every prop, function, and state is typed.
• Component-first architecture. Every UI element is a reusable component in 
/components. No one-off inline JSX blocks.
• Design tokens only. No hardcoded hex colors in component files. Always use CSS 
variables or Tailwind config tokens.
• Mobile-first CSS. Write mobile styles first, add desktop overrides with md: and lg: 
breakpoints.
• No inline styles. All styling through Tailwind classes or CSS modules.
• Form validation with Zod schemas. Every form has a schema. Validation happens on 
client AND server.
• Error boundaries on every major section. The page never white-screens on a 
component error.
• Loading states everywhere. Every async operation has a loading state. No 'hanging' 
UI.
• Accessibility first. Run axe-core in development. Fix all critical violations before any 
PR merge.
• Performance budget. Bundle size checked on every build. No new dependency 
without justification.
• Comments in English. Complex logic must have inline comments. Component 
purpose in JSDoc.
• Git commits follow Conventional Commits format: feat:, fix:, chore:, docs:, style:
